# d2 (diagrams)

## Summary
D2 (Terrastruct) is a text-to-diagram scripting language (DSL). You write declarative `.d2` source describing shapes, connections, and containers; the compiler lays it out and renders to SVG (default), PNG, or PDF. The primary deliverable is a self-contained **SVG**. Diffable plaintext source makes D2 well-suited to version-controlled architecture/system diagrams. It ships multiple layout engines (dagre default, ELK, and the paid TALA). Implemented in Go; the native CLI is the canonical interface, with an official WASM build (d2js) for JS/browser and a Go library (d2lib) for programmatic embedding.

## Toolchains
| lang | install | invoke |
| --- | --- | --- |
| shell/native | `brew install d2` (or `curl -fsSL https://d2lang.com/install.sh \| sh -s --`) | `d2 input.d2 out.svg` — flags: `--layout dagre\|elk`, `--theme N`, `--dark-theme N`, `--sketch`, `--pad`, `d2 fmt`, `d2 --watch` |
| go | `go get oss.terrastruct.com/d2` | `d2lib.Compile(...)` then `d2svg.Render(...)` — same engine as the CLI (d2lib/d2compiler/d2graph) |
| js/ts | `npm install @terrastruct/d2` | `const d2=new D2(); const r=await d2.compile(src); const svg=await d2.render(r.diagram)` — official WASM build (browser + Node) |

## Artifact kind
**svg** — the default deliverable is a standalone SVG that renders directly in the universal artifact shell. PNG/PDF are also available via output extension but go through the bundled headless render path.

## Validation
- **install:** `brew install d2` (or `curl -fsSL https://d2lang.com/install.sh | sh -s --`)
- **smoke:** `printf 'a -> b -> c\n' > /tmp/test.d2 && d2 /tmp/test.d2 /tmp/test.svg && ls -la /tmp/test.svg`
- **expect:** Exit 0; `/tmp/test.svg` created (valid SVG with three boxes a->b->c). Headless on macOS — no browser/display needed; rendering is pure Go. `d2 version` confirms install.

## Wrapper params
Key params worth exposing:
- `--layout` — `dagre` (default) | `elk` (best for orthogonal/architecture) | `tala` (paid/closed).
- `--theme` / `--dark-theme` — numeric theme IDs.
- `--sketch` — hand-drawn look.
- `--pad` — padding in px.
- Output format inferred from extension (`.svg`/`.png`/`.pdf`; PNG/PDF need the bundled headless render path).
- `--animate-interval` — multi-board animated SVG.
- `d2 fmt` — canonical formatting of source.

## Component / explorer notes
The default deliverable is a standalone SVG that renders in any image/SVG artifact shell — no special explorer required. A richer viewer adds value only for interactive features: D2 supports multi-board diagrams, layers, scenarios, and animated transitions (`--animate-interval` emits an animated SVG); those benefit from an HTML wrapper with pan/zoom. Static architecture diagrams: plain SVG kind is sufficient.
